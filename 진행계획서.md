## 1. 프로젝트 개요

### 1.1 배경

차세대 반도체 미세공정은 0.33 NA EUV의 한계(약 13 nm half-pitch)를 넘기 위해 NA 0.55의 High-NA EUV로 이행하고 있다. 이는 단순한 NA 증가가 아니라 다음 연쇄 trade-off를 동반한다.

```
NA ↑  →  Resolution ↑ (→ k₁·λ/NA)
       │
       ├─→ DOF ↓ (→ k₂·λ/NA²)
       │       └─→ wafer topology 민감도 ↑
       │
       ├─→ Mask CRA ↑  →  Mask 3D 효과 ↑
       │       └─→ shadowing, BF shift, telecentricity 손실
       │
       ├─→ Polarization 차이 ↑  (s/p 분리 모델 필요)
       │
       └─→ Aberration tolerance ↓  (wavefront 제어 정밀도 ↑)
```

이 복합 영향을 통합적으로 해석하려면 **모듈별로 분리되지 않은 시뮬레이터**이 필요하다. 본 프로젝트는 공개 학술·산업 문헌 21편 + 자체 정리한 물리 핸드북·심층 보고서를 기반으로, 위 trade-off를 코드로 재현하고 OPC/ILT/SMO/PMWO 보정의 design space를 탐색하는 시뮬레이터를 개발한다.

### 1.2 목적

본 프로젝트는 **스터디 목적**이다. 4가지 활동의 우선순위는:

1. **학습 (Understand)** ★ 1차 — 각 Phase의 의미, 흐름, 좋은 결과 기준을 정성적으로 파악한다.
2. **시각화 (Visualize)** ★ 1차 — 3D notebook으로 결과를 입체적으로 확인한다.
3. **탐색 (Explore)** — NA, sigma, dose, focus 등 입력 변수 sweep으로 거동을 관찰한다.
4. **공유 (Share)** — 21편 논문의 핵심 거동을 코드로 명시화한다. 목표는 정성 일치다.

**5번째 목적이 아닌 것**: 산업 grade 정량 검증, 실제 장비 1:1 재현.
데이터와 논문 출처가 제각각이라 환경 통일 자체가 불가능하므로 추구하지 않는다.

### 1.2.1 운영 원칙: 일단 구현 우선

엄격한 검증으로 진척이 막히면 의도적으로 단순화한다. 단순화 4중 기록 미달,
mypy strict 미적용, depth absorption paper 정량 비교 부재 등은
**study-grade에서는 P3 이하 backlog**로 둔다. 깨진 테스트, 부호 컨벤션,
단위 혼용, 감사 로그 부정직처럼 학습을 망치는 항목은 P0/P1로 유지한다.

### 1.3 비전 (3년 후의 모습)

- 외부 연구자가 `pip install`만으로 simulator를 설치하고, 5분 안에 첫 aerial image를 만들 수 있다.
- 단위 테스트 suite에서 `pytest tests/`만 돌리면 21편 논문의 핵심 정성 결과가 자동 검증된다.
- Phase 6 SMO 모듈은 differentiable PyTorch/JAX 백엔드를 지원해, ILT/PMWO를 gradient descent로 풀 수 있다.
- Hyper-NA (NA ≥ 0.75) 확장 옵션이 코드 한 줄로 활성화된다.

## 2. 추진 전략

### 2.1 4가지 운영 원칙

#### P1. 단계적 정밀화 (Progressive refinement)
각 모듈은 가장 단순한 형태로 먼저 작동하게 하고, 그 다음 정밀화한다. 예: resist는 threshold → blur → depth → stochastic의 4 level. 정밀화 단계마다 이전 level의 단위 테스트가 그대로 통과해야 한다.

#### P2. 논문 → 단위 테스트 (Paper-to-test)
논문의 정성적 결과를 단위 테스트로 변환한다. 예: paper 12 "Mask 3D 6 효과"는 6개의 단위 테스트가 된다. 이렇게 하면 코드의 어느 부분이 어느 논문의 어느 결과를 재현하는지 추적 가능하다.

#### P3. 단순화의 명시화 (Explicit simplification)
모든 단순화 가정은 코드 주석과 README에 **명시적으로 기록**한다. 예: `# Kirchhoff thin-mask 가정. 0.55 NA에서 paper 12, 17에 따라 정량 오차 가능. Phase 4에서 boundary correction 추가 예정.`

#### P4. 가역적 추상화 (Reversible abstraction)
모든 모듈은 **interface로 추상화**되어, 더 정밀한 모델로 교체 가능해야 한다. 예: `MaskModel` interface — `KirchhoffMask`, `ThinMask3D`, `RCWAMask` (미래 확장)가 동일 interface를 구현한다.

- Phase 6은 마지막 — 모든 forward 모델이 안정된 뒤에야 inverse 최적화가 의미를 가진다.

## 10. 의사결정 / 정책 (Decision Log)

본 프로젝트의 핵심 의사결정과 그 근거를 기록한다.

### D1. Python (numpy + scipy) 선택
- **결정**: 1차 구현 언어로 Python 선택
- **근거**: 학습 곡선 낮음, 학계·산업에서 표준, JAX/PyTorch 확장 용이
- **대안**: MATLAB (라이센스 부담), C++ (개발 속도 느림), Julia (생태계 부족)

### D2. Kirchhoff thin-mask baseline
- **결정**: Phase 1–3에서는 Kirchhoff thin-mask 사용. Phase 4에서 boundary correction 추가
- **근거**: 즉시 시작 가능, paper 9의 fast imaging model과 호환
- **대안**: 처음부터 RCWA → 데이터 부재 + 계산량으로 부적합

### D3. 마일스톤 기반 (시간 기반 X)
- **결정**: 일정은 weeks/months 단위 deadline 없이 entry/exit criteria로 관리
- **근거**: 사용자 명시 ("기간 제한 없음"). 학습/연구 프로젝트 성격에 부합
- **대안**: Gantt 일정 → 단독 개발 시 부담만 됨

### D4. Threshold resist를 MVP로 채택
- **결정**: Phase 5 MVP는 threshold resist만 사용. 정밀화는 별도 Phase
- **근거**: end-to-end pipeline을 빨리 닫는 것이 우선. Stochastic은 무거움
- **대안**: 처음부터 stochastic → 1차 결과 도출까지 시간 과다

### D5. anamorphic 컨벤션 fix 시점
- **결정**: Phase 1 시작 전 paper 19 figure로 4× scan / 8× cross-scan 좌표 fix
- **근거**: 한번 정하면 모든 모듈에 영향. 변경 비용 매우 큼
- **대안**: 도중 변경 → 모든 단위 테스트 재작성 위험

### D6. 단순 layout 1개로 SMO 수렴 시연
- **결정**: Phase 6 KPI K5는 "1개 layout에서 수렴"으로 한정
- **근거**: 일반화는 본 프로젝트 범위 초과. testbed 성격 강조
- **대안**: 다양한 layout → out-of-scope, 후속 연구

### 11.3 제한 사항 (반드시 명시)
- 정량 결과는 ±수십 % 오차 가능 — "근사 시뮬레이터"
- 실제 장비 imaging의 detailed prescription 재현 불가
- multilayer mirror, polarization, thermal 효과는 단순화
